# Настройка Claude структуры для проекта

Эта структура помогает Claude Code не деградировать при длительной разработке.

---

## 🚀 Установка (одна команда)

```bash
bash .claude/scripts/setup-hooks.sh
```

Это подключит:
- ✅ Pre-push hook с dumb-checks
- ✅ Post-commit hook с sync_claude_md.sh (если существует)

---

## 📂 Что установилось

```
.claude/
├── SETUP.md (этот файл)
├── memory/
│   ├── architecture-decisions.md (архитектурные решения)
│   ├── integration-contracts.md (контракты между модулями)
│   ├── tasks-completed.md (архив решённых задач)
│   └── known-issues.md (когда добавляем новые баги)
├── rules/
│   └── forbidden-patterns.md (запрещённые паттерны)
└── scripts/
    ├── dumb-checks.sh (проверка перед пушем)
    └── setup-hooks.sh (установка хуков)
```

---

## 📖 Как пользоваться

### Новая задача

1. Открыть `.claude/memory/tasks-completed.md`
2. Добавить задачу в "Текущие задачи"
3. После завершения переместить в "Архив завершённых"

### Новое архитектурное решение

1. Добавить в `.claude/memory/architecture-decisions.md` раздел
2. Объяснить **почему** это решение, **что отклонено** и почему
3. Указать файлы и номера строк

### Новый баг

1. Добавить в `.claude/memory/known-issues.md` (создать если нет)
2. Описать как воспроизвести
3. Указать временное решение если есть

### Запрещённый паттерн

1. Если найден новый паттерн который мешает → добавить в `.claude/rules/forbidden-patterns.md`
2. Добавить проверку в `.claude/scripts/dumb-checks.sh`

## 🔍 Проверка перед пушем

```bash
bash .claude/scripts/dumb-checks.sh
```

Блокирует:
- ❌ console.log в исходниках
- ❌ Hardcoded цвета вместо var(--color)
- ❌ Несинхронизованные _cfg версии
- ⚠️ Вложенные ternary (warning)
- ⚠️ Функции > 200 строк (warning)

---

## 🎯 Принцип работы

1. **CLAUDE.md** (144 строк) — только навигация, агент прочитает полностью
2. **memory/** — файлы памяти, не читаются автоматически, но доступны для контекста
3. **rules/** — правила и ограничения
4. **scripts/** — автоматизация (хуки, проверки)

Это предотвращает:
- 🔴 Контекстуальную амнезию (memory файлы)
- 🔴 Единый стиль (dumb-checks.sh блокирует)
- 🔴 Процессные пробелы (задачи в tasks-completed.md)

## ⚙️ Кастомизация

### Добавить новый dumb-check

1. Открыть `.claude/scripts/dumb-checks.sh`
2. Добавить блок (pattern см. в файле)
3. Проверить: `bash .claude/scripts/dumb-checks.sh`

### Изменить правила

1. Отредактировать `.claude/rules/forbidden-patterns.md`
2. Обновить dumb-checks.sh под новое правило
3. Сделать git commit

---

## 🔄 Отката структуры

Если что-то не нравится, легко откатить:

```bash
# Отката на предыдущий коммит
git reset --hard HEAD~1

# Или просто удалить .claude директорию
rm -rf .claude/
git checkout CLAUDE.md
```

Структура находится в отдельной ветке `claude/restructure-claude-md`, поэтому:
- ✅ Легко создать PR и обсудить
- ✅ Легко откатить если не нравится
- ✅ Легко изменить и переслать
