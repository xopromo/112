## Дайджест

### Поиск 1 — SQN и соседние метрики
Ключевой источник: «Advanced Trading Metrics: Sharpe, Sortino, Calmar, SQN & K-Ratio [2026]»
(январь 2026, https://quantamentaltrader.substack.com и villahermos.com)

- **Sortino Ratio** = pnl / downside_deviation, где dd = sqrt(mean(min(r_i,0)²))
  Отличие от Sharpe: штрафует ТОЛЬКО за отрицательную волатильность.
  Компания Zephyr Associates использует Sortino как основной показатель.

- **SQN** (Van Tharp) = (mean_trade / std_trade) * sqrt(n_trades)
  Requires per-trade PnL array. SQN>5=excellent, 3-5=good, 1-3=average.
  «The better the SQN, the easier it is to use position sizing strategies.»

- **K-Ratio** (Zephyr 1996) = slope_of_log_equity / std_error_of_slope
  Measures CONSISTENCY of equity growth — punishes erratic equity curves
  even if total PnL is the same. Purely computable from equity curve.

- **Omega Ratio** = Σ(max(r-T,0)) / Σ(max(T-r,0)), T=0
  «Most comprehensive — considers entire return distribution, no normality assumption.»
  Essentially profit factor at bar level.

### Поиск 3 — Bootstrap / Permutation Test
Ключевые источники: QuantProof, RustyBT Documentation

- **MC Permutation Test**: перемешиваем порядок сделок N=1000 раз, считаем
  p-value = доля случайных equity > actual equity.
  Отличает «скилл» от «удачи последовательности». RustyBT реализует в Rust.

- Requires per-trade PnL array — та же зависимость что и SQN.
  Если добавить `trades: [{pnl}]` в backtest() return → оба SQN и MC permutation test работают.

## Три новые гипотезы

### Гипотеза A: Sortino Ratio (★★★★★ легко, ценно)
**Оценка:** 9/10 | **Сложность:** низкая (как UPI)

```
Sortino = pnl / downside_dev
downside_dev = sqrt( mean( min(Δeq_i, 0)² ) )
```

Только `equity[]` — никаких изменений в backtest().
Отличие от существующих метрик:
- Calmar: pnl / maxDD → штрафует ОДИН момент
- UPI: pnl / ulcerIdx → штрафует длительность всех просадок
- **Sortino: pnl / downside_vol → штрафует НЕСТАБИЛЬНОСТЬ потерь**

Интерпретация: Sortino > 2 = хорошо, > 3 = отлично.
Реализация: ~15 строк, добавить в results.push() и _statsRow колонку.

### Гипотеза B: K-Ratio (★★★★☆ средне, ценно)
**Оценка:** 8/10 | **Сложность:** средняя (линейная регрессия equity)

```
K-Ratio = b1 / se(b1)
b1 = slope of OLS on log(equity[i]) ~ i
se(b1) = standard error of slope
```

Интерпретирует РАВНОМЕРНОСТЬ роста:
- Два портфеля с одинаковым итоговым PnL, но разными equity curves получат разный K-Ratio
- Высокий K-Ratio = стратегия растёт стабильно, без «горбов» и «провалов»
- Zephyr: K > 2.0 = excellent consistency

Нужен OLS (~25 строк JS). Только equity[].

### Гипотеза C: SQN + per-trade array (★★★★☆ средне, высокий потенциал)
**Оценка:** 8/10 | **Сложность:** средняя (изменение backtest() return)

Добавить `trades: [{pnl}]` в return объект backtest() (core.js).
Это разблокирует сразу две фичи:
1. **SQN** = (mean_trade / std_trade) * sqrt(n)
2. **MC Permutation Test** (будущая задача): shuffle trade order 1000x → p-value

Без trade array SQN реализовать правильно нельзя.
С trade array — вся строка mc permutation теста становится тривиальной.
