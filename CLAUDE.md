# Правила работы с проектом

## Структура файлов

Проект собирается из исходников через `python3 build.py` в единый `USE_Optimizer_v6_built.html`.

**Исходные файлы (редактировать только их):**
- `ui.js` — весь UI, таблицы, OOS-логика, рендеринг
- `opt.js` — логика оптимизации, бэктест, OOS-расчёты (_attachOOS)
- `core.js` — ядро (индикаторы, вспомогательные функции)
- `shell.html` — HTML-скелет, стили, кнопки
- `pine_export.js` — экспорт в Pine Script

## ВАЖНО: запрещено

- **Никогда не редактировать `USE_Optimizer_v6_built.html` напрямую** — он генерируется автоматически и все правки в нём будут потеряны при следующей сборке
- **Никогда не читать `USE_Optimizer_v6_built.html`** для поиска кода — искать нужно в исходниках

## Рабочий процесс

1. Найти нужный код в `ui.js`, `opt.js`, `core.js` или `shell.html`
2. Внести правки в исходный файл
3. Запустить сборку: `python3 build.py`
4. Убедиться что сборка прошла без ошибок
