# 🧬 Strategy Synthesis Engine — Руководство

## Что такое Synthesis?

**Обратный инжинириг торговых стратегий** — автоматический синтез оптимальных торговых стратегий через TPE (Tree-structured Parzen Estimator), максимизируя метрики качества.

Вместо того чтобы проверять готовые стратегии:
```
Стратегия → Metrics (проверка)
```

Synthesis делает наоборот:
```
Metrics (цели) → Optimizer → Стратегия (синтез)
```

### Главная идея
**Найти 1000-5000 оптимальных стратегий** из огромного пространства параметров, максимизируя несколько метрик одновременно (GT-Score, Sortino, Sig%).

## Параметры

### Пространство поиска — что варьировать?

| Параметр | Что делает |
|----------|-----------|
| **Entry методы** | Выбирает между Pivot, Engulfing, Pin Bar, Bollinger, ATR Breakout и т.д. |
| **Фильтры** | Выбирает какие фильтры включить (powerset: 2^N комбинаций) |
| **Параметры фильтров** | Варьирует период MA, порог ADX, пороги RSI и т.д. |
| **Exit механизмы** | Выбирает Break Even, Trailing, Reverse Signal, Time Exit и т.д. |
| **SL/TP** | Варьирует размеры stop loss и take profit |
| **Risk параметры** | Варьирует ATR период, commission и т.д. |

**Рекомендация:** Включить ВСЕ, система сама поймёт что важно.

### Целевые метрики — максимизировать

| Метрика | Что означает |
|---------|-------------|
| **GT-Score** | Anti-overfitting score (защита от переобучения) |
| **Sortino** | Downside volatility (стабильность, без положительной волатильности) |
| **Sig%** | Statistical significance (статистическая значимость) |

**Выберите все три** для баланса.

### Ограничения (Constraints) — фильтры качества

```
Min trades: 10       # минимум 10 сделок в периоде
Max DD%: 50          # максимум 50% просадки
Min WR%: 0           # выигрыши/все сделки
Min Sig%: 0          # p-value статистики
```

**Эти ограничения отсеивают плохие стратегии на лету.**

### TPE параметры

| Параметр | Значение | Что означает |
|----------|----------|------------|
| **Target** | 1000 | Сколько стратегий найти |
| **Max итераций** | 50000 | Лимит попыток поиска |
| **Exploration (γ)** | 0.25 | 0.1=exploit, 0.9=explore |

**Рекомендации:**
- **1000 стратегий** — быстрый поиск лучших за 10-20 минут
- **5000+ стратегий** — полный анализ за 60-120 минут
- **Exploration 0.25** — баланс между исследованием и использованием

### Веса метрик для ranking

```
GT-Score:    0.5  (50%) — главная цель
Sortino:     0.3  (30%) — стабильность
Sig%:        0.2  (20%) — значимость
```

Меняйте веса чтобы получить разные наборы стратегий:
- **GT=1.0, Sort=0, Sig=0** → максимальный GT-Score
- **GT=0.3, Sort=0.5, Sig=0.2** → более консервативные (стабильные) стратегии
- **GT=0.4, Sort=0.4, Sig=0.2** → универсальный баланс

## Результаты

### Таблица найденных стратегий

После завершения будет показана таблица **ТОП-1000 стратегий**, отсортированная по взвешенному скору.

**Колонки:**
- **#** — номер
- **Название** — название конфига (кликни для деталей)
- **PnL** — прибыль в пунктах
- **WR** — win rate (%)
- **n** — количество сделок
- **DD** — максимальная просадка (%)
- **GT-Score** — анти-переобучения метрика (зелёный ≥5)
- **Sortino** — стабильность
- **Sig%** — статистическая значимость (%)
- **Pareto** — ✓ если недоминируемая стратегия

### Парето фронтир

Результаты включают **Pareto frontier** — множество недоминируемых стратегий.

**Стратегия A доминирует B если:**
- A.GT ≥ B.GT И A.Sortino ≥ B.Sortino И A.Sig ≥ B.Sig
- И хотя бы одно строгое неравенство

**Стратегии на Pareto — это реальные компромиссы**, не ложные оптимумы.

## Советы и трюки

### 🚀 Ускорить поиск
- Уменьшить Target (500 вместо 1000)
- Увеличить Exploration (0.5-0.7)
- Включить меньше параметров для варьирования

### 🎯 Лучшее качество
- Увеличить Target (2000-5000)
- Уменьшить Exploration (0.1-0.2)
- Включить строгие constraints (min_dd=30%, min_wr=55%)

## FAQ

### Q: Synthesis ищет очень долго (>30 минут)
**A:** Это нормально если Target большой (5000+) или constraints строгие.
- Попробуй уменьшить Target до 1000
- Увеличь constraints (max_dd=60%, min_trades=5)

### Q: Почему результаты очень разные при разных запусках?
**A:** TPE — стохастический алгоритм, каждый запуск немного отличается. Это нормально и даже хорошо!
- Если хочешь reproducibility — используй фиксированный seed

### Q: Какой target выбрать — 500, 1000 или 5000?
**A:** Зависит от целей:
- **500** — fast screening (~5-10 минут)
- **1000** — standard analysis (~15-30 минут) — рекомендуется
- **5000+** — deep exploration (~60-120 минут)

### Q: Как выбрать weights (веса метрик)?
**A:**
- **Консервативные:** Sort=0.6, GT=0.3, Sig=0.1
- **Агрессивные:** GT=0.7, Sort=0.2, Sig=0.1
- **Универсальные:** GT=0.5, Sort=0.3, Sig=0.2 (по умолчанию)

### Q: Synthesis vs TPE — в чём разница?
**A:**
- **TPE** — ищет лучшие параметры для ФИКСИРОВАННОЙ стратегии
- **Synthesis** — ищет НОВЫЕ стратегии из огромного пространства

## Технические детали

### Архитектура

```
synthesis.js (850 строк)
├─ StrategySpace — кодирование/декодирование параметров
└─ TPEOptimizer — Tree-structured Parzen Estimator

pareto_front.js (400 строк)
├─ ParetoFront — управление Парето фронтиром
└─ rankByStrategy — многоцелевой ranking

synthesis_ui.js (500 строк)
├─ openSynthesisModal — UI модал
├─ runSynthesis — интеграция в runOpt()
└─ renderSynthesisResults — отображение таблицы

opt.js (интеграция)
└─ runSynthesis() — главный цикл синтеза
    ├─ StrategySpace инициализация
    ├─ TPE loop (batches по 100)
    ├─ evaluateCfg (backtest + metrics)
    ├─ Pareto filtering
    └─ renderSynthesisResults (update UI)
```

### Производительность

| Параметр | Target=1000 | Target=5000 |
|----------|-------------|-----------|
| Время (типичное) | 15-30 минут | 60-120 минут |
| Backtests | ~50k | ~200k+ |
| Результаты | 1000 стратегий | 5000 стратегий |
| Парето фронтир | ~200-300 | ~500-800 |
| Память | ~100-200 MB | ~500-800 MB |

**Оптимизации:**
- LHS sampling для инициализации (первые 500 итераций)
- Pareto filtering на лету (отсеивает плохие стратегии)
- resultCache (избегает переврасчёта одинаковых конфигов)
- IS/OOS split для валидации
